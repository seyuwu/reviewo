# UI / UX Guidelines

## Главная цель

Главная цель интерфейса — максимально быстро позволить пользователю найти объект и выразить свое мнение.

Интерфейс никогда не должен отвлекать пользователя от основного действия.

---

# Главные принципы

## Простота

Любой экран должен быть понятен без обучения.

Пользователь не должен задумываться над тем, куда нажимать.

---

## Минимум действий

Любая задача должна выполняться минимальным количеством действий.

Например

Поиск

↓

Выбор объекта

↓

Оценка

↓

Готово

---

## Последовательность

Все элементы интерфейса должны вести себя одинаково.

Например:

- одинаковые кнопки;
- одинаковые модальные окна;
- одинаковые формы;
- одинаковые карточки;
- одинаковые уведомления;
- одинаковые анимации.

---

# Главная страница

Главный элемент —

строка поиска.

```
Что хотите оценить?
```

Она всегда находится в центре внимания.

---

# Создание нового объекта

Если объект найден —

показываются результаты.

Если нет —

показывается

```
Создать новую страницу
```

После нажатия открывается максимально короткая форма.

Заполняются только необходимые поля.

---

# Страница объекта

Каждая страница должна иметь одинаковую структуру.

```
Header

↓

Rating Card

↓

Trust

↓

Community Tags

↓

Statistics

↓

Reviews

↓

Related Objects
```

Пользователь никогда не ищет информацию по странице.

Она всегда находится в ожидаемом месте.

---

# Browser Extension

Карточка расширения должна занимать минимум места.

По умолчанию отображаются только:

- рейтинг;
- надежность;
- лучший комментарий;
- кнопка "Подробнее".

---

# Дизайн

Используется единая дизайн-система.

Определяются:

- основные цвета;
- типографика;
- размеры;
- радиусы;
- отступы;
- тени;
- состояния компонентов.

Все новые компоненты должны использовать существующие токены дизайна.

---

# Компоненты

Каждый компонент должен быть:

- переиспользуемым;
- независимым;
- предсказуемым;
- небольшим.

---

# Адаптивность

Все страницы поддерживают:

- Desktop;
- Tablet;
- Mobile.

Никаких отдельных мобильных версий сайта.

---

# Accessibility

Минимальные требования:

- навигация с клавиатуры;
- контрастность;
- aria-атрибуты;
- поддержка screen readers;
- видимый focus.

---

# Производительность

Интерфейс должен ощущаться быстрым.

Используются:

- lazy loading;
- skeleton loading;
- optimistic updates;
- кеширование;
- виртуализация длинных списков.

---

# Обратная связь

Любое действие пользователя должно иметь мгновенный отклик.

Например:

Оценка

↓

Анимация

↓

Toast

↓

Обновление рейтинга

---

# Ошибки

Ошибки должны объяснять проблему и предлагать решение.

Не использовать сообщения вида:

"Unknown error"

или

"500"

---

# Главный UX-принцип

Пользователь никогда не должен задумываться о том, как работает система.

Он должен думать только о том, что хочет оценить.

Если интерфейс заставляет пользователя думать о платформе, а не об объекте оценки — значит, интерфейс требует упрощения.
