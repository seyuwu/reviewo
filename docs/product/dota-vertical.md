# Dota Vertical — Product

Актуально на **2026-07-16**. Первый vertical Opinia для репутации людей. Техническая спецификация: [RFC 0015](../11-rfc/0015-person-entities-and-quality-confirmations.md).

**Хосты:** публичный вход — `https://dota.opinia.ru` (тот же Next, что `opinia.ru`). С apex пути `/dota*` канонически редиректят на `dota.opinia.ru`. Локально — `http://localhost:3001/dota...` без поддоменов. Shared login и CORS: [development-workflow.md](../development-workflow.md#хосты-games--dota-важно-при-разработке).

## Позиционирование

**Не** «поиск тиммейтов». **Да** «профиль дотера с подтверждениями от тиммейтов».

One-liner для чата:

> Профиль дотера — тиммейты ставят галочки: не токсик, есть мик, норм колл. Не стата, не LFG.

## Реальность запуска

- **Каналы:** чат в Dota + ЛС в Discord. Без массовых каналов, Product Hunt, своего Discord-сервера.
- **Темп:** ручной обход — 5–15 контактов в день, недели на первые 20–30 профилей.
- **Главная задача MVP:** ссылка из чата/ЛС открывается, выглядит нормально (OG), confirm <30 сек, создатель понимает что делать дальше.
- **Вирусный loop — бонус, не опора.** На старте confirm'ы в основном от тех, кому сам напишешь «подтверди».

## User flows

### Создание профиля

1. Заход на `dota.opinia.ru` (или `opinia.ru/dota` → редирект на dota-хост)
2. «Создать профиль» → регистрация/логин
3. Wizard: Dota Account ID, MMR, роли, сервер, мик
4. Получает `dota.opinia.ru/dota/{slug}` (path тот же)
5. Экран «0/3 подтверждений» + три кнопки copy

### Подтверждение (без регистрации)

1. Друг открывает `dota.opinia.ru/dota/{slug}/confirm`
2. Видит галочки качеств
3. Нажимает «Подтвердить» — один POST
4. Footer: «Хочешь такой же профиль?» → `/dota/create`

### Поиск по Dota ID

1. Игрок знает только цифры из клиента
2. `dota.opinia.ru/dota/id/123456789` → редирект на профиль
3. Или поле «введи Dota ID» на лендинге `/dota`

## Lean GTM

### Скрипты для Dota-чата

После катки или в party chat (не all chat каждую катку):

- «сделал профиль с подтверждениями от тиммейтов, киньте галочки если играли со мной: [ссылка]»
- «кто со мной в пати — подтвердите тут, 10 сек: [ссылка на /confirm]»

### Скрипты для Discord ЛС

- «запустил штуку — профиль с галочками от тех с кем играл. можешь подтвердить? [ссылка]»
- После confirm: «спасибо. если хочешь свой — dota.opinia.ru»

### Возражения

| Возражение | Ответ |
| ---------- | ----- |
| «Зачем?» | «Чтобы не объяснять в чате что ты норм — ссылка говорит сама» |
| «Никто не подтвердит» | «Нужно 3 человека с кем реально играл — не весь сервер» |
| «Очередной профиль» | «Тут не стата — только то что друзья подтвердили руками» |

### Цели на 2–4 недели

| Неделя | Цель | Сигнал «работает» |
| ------ | ---- | ----------------- |
| 1 | 5 профилей (свой + 4 друга) | Хотя бы 1 confirm без пинга |
| 2 | 15 профилей, ~30 confirm | Кто-то сам скинул ссылку в чат |
| 3–4 | 30 профилей | 1 confirmer создал свой профиль |

Если к концу недели 2 ноль органических confirm — проблема в copy или friction, не в «мало рекламы».

## Этапы продукта

### Стадия 1 — Репутация (MVP)

- Person entity + профиль
- Dota Account ID
- Quality confirmations (green flags)
- Share flow 0/3
- OG previews

**Цель:** заставить создателя привести 3 друзей на confirm.

### Стадия 2 — Активность

- LFG «ищу игру» с auto-expire
- lastSeenAt
- Уведомления (email/in-app)

**Цель:** повод возвращаться перед вечерней каткой.

### Стадия 3 — Сеть

- Поиск по MMR/ролям
- Фильтры
- Steam OAuth + verified Dota ID

**Цель:** новые пользователи находят старых.

## MVP checklist

- [ ] `POST /dota/profiles` создаёт person с `vertical=dota`
- [ ] Dota ID уникален среди dota-профилей
- [ ] Confirm без регистрации работает
- [ ] Self-confirm заблокирован
- [ ] Прогресс 0/3 на странице профиля
- [ ] Три copy-кнопки (профиль, confirm, ID)
- [ ] OG preview в Discord ЛС
- [ ] `/dota/id/{accountId}` редиректит на профиль
- [ ] Поиск по Dota ID на лендинге

## Analytics (минимальный трекинг)

События для product analytics (или ручного учёта через логи):

| Event | Когда |
| ----- | ----- |
| `dota_profile_created` | После успешного `POST /dota/profiles` |
| `dota_share_copied` | Клик copy (тип: profile / confirm / id) |
| `dota_confirmation_submitted` | Успешный POST confirm |
| `dota_confirmer_signup_started` | Клик «создай свой» на confirm page |

Опционально: `?ref={slug}` на confirm-ссылке для атрибуции.

GA4/GTM — только если уже есть в проекте; иначе таблица + API logs на первый месяц.

## Что НЕ в MVP

- Поиск по MMR/ролям
- LFG «ищу игру»
- Red flags
- Текстовые отзывы
- Steam OAuth / OpenDota
- Email уведомления
- Ежедневные кредиты

## OG requirements

Discord ЛС показывает preview ссылки. OG image должен содержать:

- Ник игрока
- MMR (если указан)
- Top 2–3 quality counts
- «Opinia Dota» branding

Плохое OG = не кликают в ЛС. **Blocker для Discord-канала.**
