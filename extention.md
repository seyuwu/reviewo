# Browser Extension Architecture

## Назначение

Browser Extension — главный способ взаимодействия пользователя с платформой во время просмотра интернета.

Расширение должно позволять получать информацию о любой открытой странице, не заставляя пользователя переходить на сайт вручную.

Основная идея:

> Пользователь открывает страницу.

↓

Расширение автоматически определяет объект.

↓

Показывает рейтинг.

↓

Пользователь может поставить оценку буквально за несколько секунд.

Расширение должно быть максимально быстрым, ненавязчивым и практически незаметным.

---

# Главные цели

Расширение должно решать четыре задачи.

## 1. Автоматически определять объект

При открытии страницы расширение определяет:

- URL
- домен
- тип страницы
- название
- изображения
- дополнительные метаданные

После этого выполняется запрос к API.

Если объект найден —

отображается информация.

Если нет —

предлагается создать страницу.

---

## 2. Показывать информацию

Если объект существует, пользователь должен увидеть:

- рейтинг
- надежность рейтинга
- количество оценок
- лучший комментарий
- основные теги
- кнопку "Подробнее"

Все остальное открывается только при желании пользователя.

---

## 3. Позволять быстро оценить

Оценка должна занимать минимум времени.

Идеальный сценарий:

★★★★☆

↓

Спасибо!

Никаких дополнительных окон.

Комментарий — необязателен.

---

## 4. Быстрый переход

Любой объект можно открыть на сайте платформы.

Там доступны:

- история рейтинга
- все отзывы
- статистика
- похожие объекты
- дерево объектов

---

# Основные сценарии

## Сценарий 1

Пользователь открывает YouTube.

↓

Расширение определяет видео.

↓

Показывает рейтинг.

↓

Пользователь ничего не делает.

---

## Сценарий 2

Пользователь открывает неизвестную страницу.

↓

API отвечает

Not Found

↓

Расширение предлагает

Создать страницу.

↓

После подтверждения открывается форма с уже заполненными данными.

---

## Сценарий 3

Пользователь ставит оценку.

↓

API сохраняет оценку.

↓

Карточка обновляется.

---

# Принципы UX

Расширение никогда не должно мешать просмотру страницы.

По умолчанию отображается только компактная карточка.

Например

★★★★☆ 4.8

Нажатие раскрывает подробности.

---

# Настройки пользователя

Пользователь может изменить:

- положение карточки;
- время отображения;
- прозрачность;
- автоматическое скрытие;
- размер;
- светлую/темную тему;
- показывать ли лучший комментарий;
- показывать ли рейтинг автоматически;
- горячие клавиши.

Отвечает за:

# Определение объекта

Каждый сайт должен иметь собственный Parser.

Например

YouTube Parser

GitHub Parser

Amazon Parser

Steam Parser

Reddit Parser

Wikipedia Parser

...

Все Parser реализуют единый интерфейс.

```
interface PageParser {    detect()    extract()}
```

Это позволит легко добавлять поддержку новых сайтов.

# Работа с API

При открытии страницы

Content

↓

Background

↓

API

↓

Entity

↓

Ответ

↓

Карточка

Расширение не хранит бизнес-логику.

Вся логика находится на сервере.

---

# Кэширование

Для ускорения работы используется локальный кэш.

Например:

URL

↓

Entity

↓

30 минут

Это уменьшает количество запросов.

---

# Производительность

Расширение не должно:

- тормозить загрузку страницы;
- изменять DOM сайта без необходимости;
- выполнять тяжелые вычисления;
- постоянно отправлять запросы.

---

# Безопасность

Расширение:

- не собирает лишние данные;
- не читает содержимое страниц без необходимости;
- использует минимально возможные разрешения;
- работает только через HTTPS.

---

# Масштабирование

На первом этапе поддерживается Chrome.

После MVP добавляются:

- Firefox;
- Edge;
- Яндекс Браузер.

Архитектура должна позволять использовать один код для всех Chromium-браузеров.

---

# Будущее развитие

В дальнейшем расширение может получить:

- AI-краткое содержание отзывов;
- уведомления;
- быстрые коллекции;
- сравнение объектов;
- рекомендации;
- просмотр истории изменений;
- дополнительные виджеты.
